k8s部署前准备

Jul 19, 2019

node节点准备

  • 至少需要4个节点
  • 对于一个拥有10个以上节点的机器来说，apiserver和etcd可以一起安装在一台1核1GB内存的机器上
  • 合理分配内存和cpu不是所有节点都需要同样的配置

    网络准备

  • 配置网络完成pod的ip地址路由
  • 建立一个拓扑网络
  • 准备kubernetes集群所使用的集群子网
  • 给每一的node的Pod地址分配一个CIDR子网
  • 配置防火墙允许访问apiserver端口80和443
  • sysctl设置”net.ipv4.ip_forward = 1”开启ip转发

    集群命名规范

  • 提前规划好集群命名规范

    硬盘配置

  • 数据盘使用xfs并开启pquota /dev/vdb /data xfs defaults,noatime,pquota 0 0

    软件包准备

准备证书使用HTTPS方式访问apiserver

使用HTTPS访问apiserver

  • 配合电子证书和用户登录信息使用
  • 如果要用HTTPS这个方式，你需要准备电子证书和用户登录信息
    准备多个证书:
  • 主节点会是一个HTTPS服务器，所以需要一个证书
  • 如果Kuberlets需要通过HTTPS提供API服务时，这些kuberlets需要出示证书向主节点证明主从关系
    除非你决定要一个真正CA来生成这些证书的话，你需要一个根证书，并用这个证书来给主节点，kuberlet和kubectl的证书签名
  • 参见“cluster/gce/util.sh”脚本里的“create-certs”
  • 并参见“cluster/saltbase/salt/generate-cert/make-ca-cert.sh”和cluster/saltbase/salt/generate-cert/make-cert.sh“

准备登陆信息

  • 对应验证身份的令牌token或密码passwd
  • 令牌token(dd if=/dev/urandom bs=128 count=1 2>/dev/null | base64 | tr -d “=+/“ | dd bs=32 count=1 2>/dev/null)

    创建kubeconfig文件

  • 需要kuberconfig文件里添加证书，密钥和主节点IP
  • kuberconfig文件可以参照“cluster/gce/configure-vm.sh”中的代码具体如下

需要关注

配置docker容器

  • 如何配置Docker取决于你网络是基于routable-vip还是overlay-network
  • kube-proxy可以代替docker来设置iptables
  • 配置Docker可以打开文件的数目DOCKER_NOFILE=1000000

    kubelet

    所有节点都需要运行kubelet
    可参考参数:
  • 如果选择HTTPS的安全配置
    • api-servers=https://$MASTER_IP
    • kubeconfig=/var/lib/kubelet/kubeconfig
  • 否则，使用防火墙的安全配置
    • api-servers=http://$MASTER_IP
  • config=/etc/kubernetes/manifests
  • cluster-dns= 是用来配置DNS服务器的地址
  • cluster-domain= 是为DNS集群地址使用的DNS域名前缀
  • docker-root=
  • root-dir=
  • configure-cbr0=
  • register-node

    网络

  • 为了pod的网络通信，需要给每一个节点分配一个自己的CIDR网段。这个叫做 NODE_X_POD_CIDR 比如， NODE_X_POD_CIDR 是 10.0.0.0/16
  • 在初始化的脚本里，设置kubelet的选项为 –configure-cbr0=true ，并重启kubelet服务。Kubelet会自动设置cobr0. 它会一直等待，直到节点controller正确设置Node.Spec.PodCIDR
