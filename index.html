Atom as Go IDE

安装Golong

设置环境变量

  • ～/.bash_profile

  • 使修改立刻生效
    source ~/.bash_profile

  • 查看go的环境变量
    go env

  • GOROOT目录
    Go 的安装目录。这个目录是 Go 寻找内置库的地方
  • GOPATH 工作目录。
    这个目录用于存放从网上下载的第三方库以及你自己创建的项目
  • GOBIN
    是指执行 go get some-package 的时候将生成的二进制命令保存到的目录

安装Atom基础插件

a. go-plus插件

go-plus是Atom上面的一款开源的Golang开发环境插件，
项目地址:https://github.com/joefitzgerald/go-plus
他需要依赖一些的Golang工具:

  • autocomplete-go ：gocode的代码自动提示
  • gofmt ：使用goftm,goimports,goturns
  • builder-go: go-install 和go-test,验证代码，给出建议
  • gometalinet-linter: goline,vet,gotype的检查
  • navigator-godef: godef
  • tester-goo : go test
  • gorename : rename工具

在Atom中的Preference中可以找到install菜单,输入go-plus
atom安装的时候会自动解决相关依赖关系

Kubernetes单节点部署

官方提供了Kubernetes部署的3种方式

  • minikube
    Minikube是一个工具，可以在本地快速部署一个单节点的Kubernetes,不能用于生产环境
    官方文档:https://kubernetes.io/docs/setup/minikube/

  • kubeadm
    kubeadm可以快速部署一个kubernetes集群，kubeadm设计目的为新用户开始尝试kubernetes提供一种简单
    的方法。目前是Beta版。

    官方文档:https://kubernetes.io/docs/reference/setup-tools/kubeadm/kubeadm/
        https://kubernetes.io/docs/setup/independent/install-kubeadm/

  • 二进制包
    从官方下载发行版的二进制包，手动部署每个组件，组成Kubernetes集群。目前企业生产环境中主要使用该方式。
    下载地址:https://github.com/kubernetes/kubernetes/blob/master/CHANGELOG-1.11.md#v1113

1.安装需求

  • 操作系统
  • 内存2GB，2核cpu
  • 每个节点间可以通信
  • 每个节点主机名、mac地址、produc_uuid唯一
  • 禁止swap分区，这样子才能使kubelet正常工作

在终端root用户下分别执行：rpm -qa|grep jdk、rpm -qa|grep gcj 确定系统自带jdk版本号。
rpm -e –nodeps 后面跟上卸载的包名，卸载系统自带的jdk，执行以下命名检查卸载是否完成。
java -version

a．安装新版本JDK

以root用户把安装文件jdk1.8.0_102tar.gz拷贝到/usr/local/目录下

进行解压tar -zxvf jdk1.8.0_102tar.gz
执行chmod 755 -R jdk1.8.0_102安装包赋予相关权限
配置环境变量：vim /etc/profile 文件最后增加如下配置
export JAVA_HOME=/usr/local/jdk1.8.0_102
export CLASSPATH=$CLASSPATH:$JAVA_HOME/lib:$JAVA_HOME/jre/lib
export PATH=$JAVA_HOME/bin:$JAVA_HOME/jre/bin:$PATH:$HOME/bin
export JRE_HOME=/usr/local/jdk1.8.0_102/jre
然后执行source /etc/profile 使配置生效。
